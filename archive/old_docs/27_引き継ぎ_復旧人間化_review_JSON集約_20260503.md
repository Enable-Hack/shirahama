# タスク A: settings.production.json で復旧コマンドも deny に追加 (5 分)

## A-1. 現状の歪み

封じ込め (iptables -j DROP / kill -9 等) は deny だが、復旧操作（設定書き換え + reload）が抜け道になっている：
- `ssh root@10.1.1.1 'cp /etc/named.conf.new /etc/named.conf'` — Edit(/etc/**) deny を ssh 越し cp で迂回
- `sysctl -w net.ipv4.tcp_syncookies=1` — deny に存在せず AI が直接打てる
- `mv /tmp/named.conf /etc/named.conf` — 同様の迂回経路
- `ln -sf /tmp/evil /etc/passwd` — symlink 経由の書き換え

## A-2. 追加する deny ルール

settings.production.json の "deny" 配列に以下を追加:

  "Bash(*sysctl*-w*)",
  "Bash(*sysctl*--write*)",
  "Bash(*cp*/etc/*)",
  "Bash(*cp*/usr/local/etc/*)",
  "Bash(*mv*/etc/*)",
  "Bash(*mv*/usr/local/etc/*)",
  "Bash(*ln*/etc/*)",
  "Bash(*ln*/usr/local/etc/*)",
  "Bash(*scp*:/etc/*)",
  "Bash(*scp*:/usr/local/etc/*)",
  "Bash(*install*/etc/*)",
  "Bash(*install*/usr/local/etc/*)",

settings.json (build mode) は触らない（build 中は構築のため必要）。
_comment_deny を「封じ込め+復旧の両方を AI 実行禁止」に更新。

## A-3. 検証
