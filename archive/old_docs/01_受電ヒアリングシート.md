# 01. 受電ヒアリングシート

> 電話を取ったとき、**最初に聞くべきこと** と **聞いた内容の記録フォーマット**。
> このシートは電話応対係が印刷して手元に置き、1件1枚で使用する想定。

---

## 共通冒頭（どのパターンでも最初に）

```
「お電話ありがとうございます、管理会社の〇〇です。」
    ↓（相手の第一声を最後まで遮らず聞く）
「ご連絡ありがとうございます。状況を詳しくお伺いしてもよろしいでしょうか。
 お名前と、折り返しのお電話番号をお伺いできますか。」
```

### 必ず冒頭で確保する3項目

| # | 項目 | 記入 |
|---|------|------|
| 1 | 相手の所属・氏名 |  |
| 2 | 折り返し電話番号・メール |  |
| 3 | 受電時刻（mm/dd HH:MM） |  |

---

## パターン判定（受電の最初の30秒で分類）

相手の第一声から用件を分類します。

| 相手 | 第一声の例 | パターン |
|---|---|---|
| 一般ユーザ | 「〇〇が使えない」「画面がおかしい」「メールが届かない」 | **A. 苦情・問い合わせ** |
| 顧客担当者 | 「取引先からクレームが来た」「監視の数値が異常」 | **B. 報告・通報** |
| 顧客社長・上役 | 「どうなってる」「いつ復旧する」「誰の責任」 | **C. 苦情・要求（対経営層）** |
| 取引先・一般第三者 | 「貴社から不審なメールが来た」「貴社サイトから攻撃されている」 | **D. 外部からの通報** |

> パターン判定できない場合は迷わず **パターンA** のヒアリング項目で進める。

---

## パターン A：ユーザからの苦情・問い合わせ

> 最も頻度が高い。技術担当への引き継ぎのほとんどがここから始まる。

### A-1. 聞き取り項目

| # | 質問 | 記入例 |
|---|------|--------|
| 1 | どの端末・サービスで起きていますか | 営業部のノートPC / 社内ポータル / メール |
| 2 | いつから発生していますか | 10:15 頃から |
| 3 | どういう症状ですか（画面の文字・エラー番号・音・色） | 赤い画面、英文メッセージ |
| 4 | 直前に何をしましたか（メールの添付開封、USB接続、インストール等） | メール添付の Excel を開いた |
| 5 | 他の人にも起きていますか／同じ部署で誰かに聞きましたか | 隣の席の田中さんも同じ |
| 6 | 今、その端末で何か操作していますか | 何もしていない |
| 7 | 電源は入ったままですか | はい |

### A-2. 電話で **絶対に指示すること**

- **その端末に触らない**（キーを押さない、クリックしない）
- **電源は切らない、再起動しない**（揮発ログ保全のため）
- **ネットワークケーブルは抜かずに待機**（勝手に抜くと状況が変わる。技術担当から指示が出るまで待つ）
- **画面に出ているメッセージの写真を撮ってもらう**（可能なら）
- **折り返しこちらから連絡する時刻を約束**（例：「10分以内に折り返します」）

### A-3. 電話で **絶対に言わないこと**

- 「ランサムウェアですね」「ウイルスです」など **原因の断定**（この時点では事実確認のみ）
- 「大したことないです」「すぐ直ります」など **根拠のない安心づけ**
- 「パスワードを教えてください」 ← **絶対 NG**、本人に操作してもらう

### A-4. 記入欄

```
受電時刻：                    対応者：
発信者：      所属：       氏名：       折り返し先：
対象端末／サービス：
発生時刻（相手認識）：
症状（見える／聞こえること）：
直前の操作：
他に同様事象あり：  □なし  □あり（   ）
相手に依頼した待機事項：□端末触らない □電源切らない □写真撮影
次アクション約束時刻：
```

## パターン B：顧客担当者からの報告・通報

> 顧客側で気づいた異常の第一報。情報の精度が高いことが多い。

### B-1. 聞き取り項目

| # | 質問 |
|---|------|
| 1 | どのシステム・サービスで異常を観測しましたか |
| 2 | 観測された具体的な数値・ログ（アクセス数急増、応答遅延、エラー率等） |
| 3 | 観測時刻（できれば開始時刻と現在までの推移） |
| 4 | 顧客側で既に実施した操作はありますか（再起動／切り離し／設定変更） |
| 5 | 影響を受けている利用者の範囲（全員／一部／社外／社内） |
| 6 | 顧客側の担当者は連絡つきますか（今後の窓口確認） |

### B-2. 記入欄

```
受電時刻：                    対応者：
発信者：      所属：       氏名：       折り返し先：
観測システム：
観測時刻・推移：
観測値（数値・ログ・エラー）：
顧客側で実施済みの操作：
影響範囲：
今後の窓口：
```

## パターン C：顧客経営層からの苦情・要求

> 強い口調の電話。**落ち着いて事実を伝える** ことが最重要。

### C-1. 聞き取り項目

| # | 質問／応対 |
|---|------|
| 1 | 「ご心配をおかけしております。現在の状況を正確にお伝えします」と切り出す |
| 2 | 何をどこまで知っているか確認（顧客側の情報源：取引先／メディア／社員） |
| 3 | どんな判断を求められているか（即時復旧／公表可否／取引先通知） |
| 4 | 折り返し報告の頻度合意（例：30分ごと／大きな変化時のみ） |

### C-2. 電話で伝える4点テンプレ

```
【事実】   現時点で分かっていること：〇〇
【対応中】 今やっていること：〇〇
【見通し】 次の区切りまでの時間：〇〇
【判断事項】ご判断いただきたいこと：〇〇
```

### C-3. NG対応

- 事実が固まっていないのに **「すぐ直します」と約束する**
- 技術的詳細を長々と話す（経営層は判断材料が欲しい）
- 「担当が分かりません」と丸投げ
- **逆切れ、反論、沈黙**（代わりに「確認して5分以内に折り返します」で切る）

### C-4. 記入欄

```
受電時刻：                    対応者：
発信者：      氏名：       折り返し先：
相手が把握している事実（情報源含む）：
求められている判断：
合意した報告頻度：
次回連絡時刻：
```

---

## パターン D：外部第三者からの通報

> 「貴社サーバから不審なパケットが来ている」「貴社ドメインから迷惑メールが来た」等。**陽動の可能性も考慮** して聞き取る。

### D-1. 聞き取り項目

| # | 質問 |
|---|------|
| 1 | どのようにして弊社関与だと特定しましたか（送信元IP／ヘッダ情報／ドメイン） |
| 2 | 発生時刻・頻度・規模 |
| 3 | 可能であれば該当するログ・メール・URL を転送してもらえますか |
| 4 | 通報者の連絡先（折り返し先／メール）を再確認 |

### D-2. 判定ポイント

- **本当に自社（顧客）起因か** を、ログ照合まで断定しない
- 陽動目的の虚偽通報の可能性（別件への注意を逸らすため）もある前提で、すぐに全社対応に切り替えない
- 証拠（ログ・パケットキャプチャ・ヘッダ）の提供を依頼

### D-3. 記入欄

```
受電時刻：                    対応者：
発信者：      所属：       氏名：       折り返し先：
特定根拠（IP、ドメイン、ヘッダ等）：
発生時刻・頻度・規模：
提供された証拠：□あり（    ） □後日転送
信憑性評価：□高 □中 □低（理由：          ）
```

## 受電終了時の締めフレーズ

```
「お知らせいただきありがとうございました。
 〇〇（時刻）までに、こちらから折り返しご連絡します。
 その間、〇〇（端末操作／再起動など）は行わずお待ちください。
 何か変化がありましたら、お手数ですがこの番号にお電話ください。」
```

---

## 受電ヒアリングの基本原則（全パターン共通）

- **復唱する**：聞き取った内容を「〇〇で、△△ということですね」と声に出して確認
- **記録係が同時に書く**：電話応対係は話すことに集中し、記録係がボードに書く
- **時刻は必ず入れる**：受電／発信の全てに mm/dd HH:MM を添える
- **確証のない情報は「相手の申告」と明示**：「ユーザ申告：赤い画面表示」のように事実と申告を分ける
- **曖昧なら切って折り返す**：「確認して折り返します」は逃げではなく品質担保
- **電話の用件が複数混ざっていたら分ける**：1件=1シートを徹底
