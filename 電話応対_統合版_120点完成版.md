# 電話応対 統合版 120点完成版

対象: 情報機器管理コンテストで、顧客・利用者・CIC・第三者から電話を受ける担当者  
用途: 本番中にこの1本だけを開いて使う  
方針: 電話で解決しきらず、相手を落ち着かせ、事実を集め、技術担当へ正確に渡す

---

## 0. 最重要ルール

1. 認証情報、パスワード、秘密鍵、VPN情報、管理画面URLの認証情報はこの資料に書かない。
2. 電話口でパスワード、カード番号、マイナンバー、秘密情報を聞かない。
3. 「直りました」「漏えいはありません」「安全です」と断定しない。
4. 原因が不明でも、約束した時刻には必ず第一報・経過報告を入れる。
5. 不審電話、警察・報道・外部機関を名乗る電話では、電話口で情報を出さず、公式経路で折り返す。
6. メール侵害が疑われる場合、メールで重要情報を送らない。電話、運営指定チャネル、チケットを優先する。
7. 証拠を消す操作を相手にさせない。管理画面での削除、再インストール、ログ削除、初期化は止める。
8. 技術担当の確認前に、顧客へ復旧完了を伝えない。

## 1. 90秒電話応対カード

電話が来たら、まずこの順番だけ守る。

```text
1. 名乗る
   お電話ありがとうございます。Booth○のwhiskey、○○です。
   記録のためメモを取りながら伺います。

2. 相手と折返し先を取る
   御社名、お名前、折返し可能なお電話番号を伺ってもよろしいでしょうか。

3. 5項目を聞く
   いつからですか。
   どのサービスですか。
   どのような症状ですか。
   何名・どの範囲で発生していますか。
   直前に変更や、すでに試した対応はありますか。

4. 復唱する
   ○時頃から、○○で、○○という症状が、○○の範囲で発生している、という理解でよろしいでしょうか。

5. 緊急度を仮判定する
   P1/P2/P3/P4 のどれかを決める。迷ったら一段階高く扱う。

6. 余計な操作を止める
   原因確認が終わるまで、画面・ログ・エラーメールは削除せず保存してください。

7. 折返しを約束する
   ○分以内に、この番号へ第一報をご連絡します。
   原因が特定できていない場合でも、約束時刻には現状をご報告します。

8. 技術担当へ渡す
   【P_候補】事象 / 発生時刻 / 対象 / 範囲 / 相手 / 約束時刻 / 確認依頼
```

## 2. 役割分担

| 役割 | やること | やらないこと |
|---|---|---|
| 電話担当 | 受電、復唱、メモ、緊急度仮判定、折返し、チケット下書き | 技術調査を抱え込む、未確認の復旧宣言 |
| 技術担当 | サーバ、ネットワーク、ログ、設定、復旧確認 | 顧客説明を電話担当に無断で変える |
| 共有担当を兼ねる場合 | 共有メモ更新、時刻管理、次回連絡リマインド | 記録なしで口頭だけの引継ぎ |

2人チームの場合は、電話担当と技術担当を分ける。電話担当は「相手対応」と「時刻管理」に集中する。

## 3. 緊急度判定 P1〜P4

| 緊急度 | 条件 | 第一報 | 経過報告 |
|---|---|---:|---:|
| P1 | サービス停止、業務停止、被害拡大中、改ざん公開中、情報漏えい疑い、DNS改ざん、スパムリレー疑い | 10分以内 | 15分ごと |
| P2 | 一部利用不可、不正アクセス疑い、メール異常、限定的改ざん、影響範囲未確定 | 15分以内 | 30分ごと |
| P3 | 限定的な不具合、再現性が低い相談、操作確認、緊急性の低い調査依頼 | 30分以内 | 必要時 |
| P4 | 一般質問、資料請求、営業、取材の一次受付 | 当日中または指定窓口案内 | 原則不要 |

迷ったら一段階高く扱う。過小評価の放置は大きな減点になる。


## 6. 技術担当へのハンドオフ

電話担当は、長文ではなく次の形で渡す。

```text
【P_候補】事象名
発生: HH:MM頃
対象: サービス名 / URL / アカウント / 機器
範囲: 全体 / 一部 / 外部のみ / 内部のみ / 未確認
相手: 会社名 氏名 電話番号
症状: エラー内容、不審表示、利用不可内容
既措置: 相手が試したこと、こちらが指示したこと
依頼: 確認してほしい技術項目
約束: HH:MMまでに第一報
注意: メール侵害疑い / 報道リスク / なりすまし疑い / 証拠保全
```

例:

```text
【P1候補】Web改ざん
発生: 14:20頃
対象: user12.com1.local
範囲: 外部から閲覧可、他ユーザ影響は未確認
相手: com1 田中様 090-xxxx-xxxx
症状: トップページが英語の改ざん表示
既措置: 顧客へ画面保存、管理画面操作停止を依頼済み
依頼: Webログ、更新履歴、不審ファイル、他ユーザ影響確認
約束: 14:35までに第一報
注意: 公開改ざんのためP1扱い
```

## 7. 症状別スクリプト

### 7-1. Webサイトが見られない

聞くこと:

- 対象URL
- 発生時刻
- 内部だけか、外部からも不可か
- エラー内容
- 全員か一部か
- 直前変更の有無

電話例:

```text
ご連絡ありがとうございます。Webサイトが見られないとのこと、状況を確認します。
対象のURLを教えてください。
いつ頃から発生していますか。
社内からだけでなく、社外からも同じ症状でしょうか。
画面にはエラー番号、タイムアウト、真っ白など、何が表示されていますか。

確認します。
○時頃から○○のWebサイトが開けず、影響は○○、ということですね。
こちらでWebサーバ、DNS、ネットワーク経路を確認し、10分以内に第一報をご連絡します。
```

技術担当へ渡す確認依頼:

- Webサービス稼働
- アクセスログ、エラーログ
- DNS名前解決
- FW、ルーティング
- 外部・内部からの疎通

---

### 7-2. Webページ改ざん

聞くこと:

- 改ざんされたURL
- 初回発見時刻
- 表示内容
- 管理画面操作の有無
- 他ユーザ・他ページへの影響

電話例:

```text
ご連絡ありがとうございます。不審な表示があるとのこと、最優先で確認します。
まず、表示されている画面は閉じず、可能であればスクリーンショットを保存してください。
原因確認が終わるまで、管理画面での編集、削除、再インストールはお控えください。

どのページに表示されていますか。
いつ頃、どなたが最初に気づかれましたか。
直前に更新作業やプラグイン追加などはありましたか。

確認します。
○時頃に○○ページで不審な表示を確認された、ということですね。
こちらでWebログ、更新履歴、不審ファイル、他ユーザへの影響を確認し、10分以内に第一報をご連絡します。
```

技術担当へ渡す確認依頼:

- 該当ドキュメントルート
- 更新履歴
- Webログ
- 不審ファイル
- 他ユーザ影響
- バックアップ状況

禁止:

- 顧客に削除・初期化を指示しない
- 未確認で「戻せます」と言わない
- SNS発信や外部告知の判断を電話担当がしない

### 7-3. メールが送受信できない

聞くこと:

- 送信不可、受信不可、遅延のどれか
- 対象アカウント
- エラーメールの有無
- いつからか
- 全体か一部か
- Webやネットワークは使えるか

電話例:

```text
ご不便をおかけしております。メールの送受信について確認します。
送信と受信のどちらで問題が出ていますか。
特定の相手先だけでしょうか。それとも全体的に発生していますか。
エラーメールが返っている場合、件名や本文の冒頭を教えてください。
エラーメールは削除せず保存してください。

確認します。
○時頃からメールの○○に問題があり、影響は○○、ということですね。
こちらでメールサーバ、DNS、メールキュー、ログを確認し、15分以内に第一報をご連絡します。
```

技術担当へ渡す確認依頼:

- メールキュー
- 送受信ログ
- DNS、MX
- スパムリレー疑い
- 特定アカウントか全体か

---

### 7-4. DNS異常

聞くこと:

- 対象FQDN
- IP直打ちで到達するか
- 内部だけか外部もか
- いつからか
- 直前のDNS変更有無

電話例:

```text
名前でのアクセスに問題があるとのこと、DNSを含めて確認します。
アクセスできない名前を教えてください。
IPアドレスではアクセスできますか。
社内からだけでしょうか、社外からも同じでしょうか。
直前にDNSやサーバ設定の変更はありましたか。

確認します。
IPアドレスでは到達できるが、名前では到達できない状態ですね。
DNS設定、名前解決、経路を確認し、15分以内にご報告します。
```

技術担当へ渡す確認依頼:

- ゾーン設定
- namedまたはDNSサービス状態
- 内部・外部名前解決
- キャッシュ影響
- 上位DNS経路

### 7-5. ネットワーク不安定

聞くこと:

- 対象拠点
- 対象端末
- 内部通信か外部通信か
- 完全停止か断続的か
- いつからか
- 直前変更の有無

電話例:

```text
ネットワーク接続が不安定とのこと、影響範囲を確認します。
どの拠点、どの端末で発生していますか。
社内サーバだけでなく、外部サイトにもつながりにくい状態でしょうか。
完全につながらないのか、遅い・途切れる状態なのか教えてください。

確認します。
○時頃から○○拠点で接続が不安定、影響は○○、ということですね。
こちらでルータ、スイッチ、ファイアウォール、サーバ側状態を確認し、10分以内に第一報をご連絡します。
```

技術担当へ渡す確認依頼:

- ルータ・スイッチログ
- インターフェース状態
- FW状態
- ルーティング
- DHCP
- サーバ側負荷

---

### 7-6. 不正アクセス・情報漏えい疑い

聞くこと:

- 対象アカウント・システム
- 不審ログイン時刻
- 接続元IPなど観測事実
- 現在も続いているか
- 何を見て気づいたか
- 証拠が残っているか

電話例:

```text
不正アクセスの可能性があるとのこと、慎重に確認します。
パスワードなどの機密情報は、この電話ではお伝えにならないでください。
ログや画面は削除せず、そのまま保存してください。

該当するアカウント名と確認された時刻を教えてください。
どの画面、どのログで気づかれましたか。
現在も不審な操作が続いている様子はありますか。

確認します。
○時頃に不審なログイン履歴を確認し、情報漏えいの可能性を心配されているということですね。
緊急度を高めに扱い、ログ保全と影響範囲確認を進めます。
10分以内に第一報をご連絡します。
```

技術担当へ渡す確認依頼:

- 認証ログ
- 接続元IP
- セッション状態
- 権限変更
- ログ改ざん有無
- パスワード変更、セッション無効化、通信遮断の要否

### 7-7. 苦情・怒っている相手

方針:

- 反論しない
- 事実、謝罪、次の行動の順で話す
- 「こちらでは問題ありません」と言い切らない
- 約束時刻を短く具体化する

電話例:

```text
ご不便をおかけしており、申し訳ございません。
何度もご連絡いただいている状況を重く受け止めています。

まず、現在も同じ症状が続いているか確認させてください。
前回ご連絡いただいた時刻と、今回新たに変化した点はありますか。

こちらで過去の対応内容と現在の状況を確認し、曖昧にせず整理します。
○分以内に、現在の対応状況と次の手順をご連絡します。
```

### 7-8. 警察・公的機関を名乗る電話

方針:

- 電話口で情報を出さない
- 公式に掲載された代表番号へこちらから折り返す
- マネージャ、メンター、運営へ即時共有
- 個人情報、通信記録、顧客名簿、攻撃者IPを即時提供しない

電話例:

```text
ご連絡ありがとうございます。Booth○のwhiskey、○○です。
記録のためメモを取りながら伺います。

ご依頼の件、手続き上、お電話口での情報提供はいたしかねます。
公式に掲載されている代表番号をこちらで確認し、折り返しの形で対応します。

恐れ入りますが、ご所属、お名前、ご用件、折返し先の部署名を伺ってもよろしいでしょうか。
また、必要な情報は書面または運営指定の正式な連絡経路でご依頼ください。
```

### 7-9. 報道・取材

方針:

- 事実確認前にコメントしない
- 被害有無、顧客名、原因、攻撃者情報を話さない
- 広報・運営・責任者へ回す

電話例:

```text
お問い合わせありがとうございます。
恐れ入りますが、取材・公表に関するお問い合わせは、担当窓口で確認の上、正式に回答します。
この電話口では、個別事案の有無や内容についてお答えできません。

媒体名、お名前、ご連絡先、ご質問内容を伺ってもよろしいでしょうか。
担当へ共有し、正式な窓口から折り返します。
```

### 7-10. CIC・運営からの連絡

方針:

- まず相手と用件を確認
- 指摘内容を復唱
- 対応予定時刻を伝える
- 完了後は運営指定チャネルへ報告

電話例:

```text
ご連絡ありがとうございます。Booth○のwhiskey、○○です。
ご指摘内容を確認します。
○○について、○時頃から○○の状態ということでよろしいでしょうか。

こちらで確認し、○分以内に状況をご報告します。
必要な場合は、運営指定のチケットまたは連絡経路にも記録します。
```

## 8. 折返し電話の方法

折返しは「電話をかけるだけ」ではなく、相手確認、伝える範囲、記録まで含めて対応する。

### 8-1. 発信前チェック

- [ ] 折返し先番号が受電メモと一致している
- [ ] 相手の会社名、氏名、事象、約束時刻を確認した
- [ ] 技術担当に「今伝えてよい内容」を確認した
- [ ] 原因未特定、復旧未確認、影響範囲未確定のどれかを整理した
- [ ] メール侵害、不審電話、報道、公的機関対応の注意点がないか確認した

発信前に技術担当へ一言確認する。

```text
○○の件、これから第一報を入れます。
伝える内容は「現状○○、原因候補○○、次は○時○分」で問題ありませんか。
復旧済みとはまだ言わない認識で合っていますか。
```

### 8-2. 折返し時の名乗りと相手確認

電話がつながったら、いきなり詳細を話さない。まず相手を確認する。

```text
お世話になっております。Booth○のwhiskey、○○です。
先ほど○○の件でお電話いただいた件について、折返しのご連絡です。
念のため、○○様ご本人でいらっしゃいますでしょうか。
```

相手が本人か不明な場合:

```text
恐れ入りますが、詳細な内容をお伝えする前に、先ほど伺ったご所属とお名前を確認させてください。
確認が取れない場合は、改めて先ほどのご担当者様宛にご連絡します。
```

本人以外が出た場合:

```text
恐れ入ります。個別の内容を含むため、詳細はご本人様に直接お伝えします。
○○様へ、Booth○のwhiskeyから折返しがあった旨だけお伝えいただけますでしょうか。
```

### 8-3. 折返しで話す順番

折返しでは、次の順番で短く話す。

1. 名乗りと本人確認
2. どの件かを一言で確認
3. 現時点で分かっている事実
4. まだ断定できないこと
5. 実施中または完了した対応
6. 相手にお願いしたいこと
7. 次回連絡時刻
8. 質問の有無

基本形:

```text
先ほどの○○の件で第一報です。
○時○分時点で、○○の状態を確認しています。
現在、○○を原因候補として調査しています。
まだ○○については断定できていません。
こちらでは○○を実施中です。
お手元では、画面やエラーメールを削除せず保存してください。
次は○時○分までに経過をご報告します。
ここまででご不明な点はございますか。
```

### 8-4. 不在・留守電・つながらない場合

折返し先につながらない場合も、記録と次アクションを残す。

```text
HH:MM 折返し発信、応答なし
HH:MM 2回目発信、応答なし
HH:MM 共有メモへ記録、技術担当へ通知
次アクション: ○分後に再発信 / 運営指定チャネルへ連絡 / チケットへ経過記録
```

留守電に残してよい内容:

```text
Booth○のwhiskey、○○です。
先ほどお問い合わせいただいた件で折返しのお電話です。
詳細確認のため、改めてご連絡します。
```

留守電に残してはいけない内容:

- 障害原因
- 改ざん、不正アクセス、漏えい疑いなどの詳細
- 顧客名以外の関係者情報
- IPアドレス、アカウント名、ログ内容
- 認証情報

### 8-5. 折返し後の記録

通話後すぐに共有メモへ追記する。

```text
■ 折返し記録
発信時刻:
応答者:
本人確認: 済 / 未 / 本人以外
伝えた内容:
相手の反応・追加情報:
次回連絡時刻:
技術担当へ再共有した内容:
```

## 13. 提出答案としての要約

本コンテストの電話応対では、丁寧な言葉遣いだけでなく、インシデント対応として必要な情報を短時間で集め、技術担当へ正確に渡すことが重要である。

基本方針は次の通り。

1. 名乗り、相手と折返し先を確認する。
2. 発生時刻、対象サービス、症状、影響範囲、直前変更を聞く。
3. 内容を復唱し、認識違いを防ぐ。
4. P1〜P4で緊急度を仮判定し、迷ったら高めに扱う。
5. 証拠を消さないよう案内し、余計な操作を止める。
6. 折返し時刻を明示して調査に入る。
7. 技術担当へ簡潔にハンドオフする。
8. 原因未特定でも約束時刻に経過報告する。
9. 復旧後は原因、対処、確認方法、再発監視を報告する。
10. 全内容を記録し、最終報告と振り返りに使える形にする。

電話担当は、電話で問題を解決しきる役ではない。相手を落ち着かせ、事実を集め、約束を守り、技術担当が復旧に集中できる状態を作る役である。この分担ができているチームは、技術点だけでなく、運用品質とコミュニケーション品質でも高く評価される。
