## 演習中にやること

### トラブル対応の基本フロー
1. トラブルの内容・発生時刻を**メモ**する
2. ログを確認して原因を特定する
3. 対処を実施する
4. 復旧を確認する
5. 対応内容をメモしておく（チケット提出用）

### よくあるトラブルと確認箇所
| トラブル | 最初に確認する場所 |
|---|---|
| 外部からの不正侵入 | /var/log/messages, /var/log/rsyslog/ |
| Webページへの不正書き込み | Apacheのアクセスログ, WordPressの投稿 |
| メールの異常 | Sendmailのログ, メールキュー |
| ネットワークダウン | RTXのログ, MRTGのグラフ |
| DNSの異常 | BINDのログ, named.conf |
