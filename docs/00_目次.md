# 情報危機管理コンテスト 対応ワークブック

> 和歌山大学 情報危機管理コンテスト向け、**電話受電 → 技術引き継ぎ → シナリオ対応 → 完了報告** までを通しで運用するためのワークブック。
> 1つの PDF／Markdown にまとめず、**工程別にファイルを分割** して、4人チームが同時並行で参照できる構成にしている。

---

## 前提（大会の性質）

- 4人1チームで、**実機環境のサーバとネットワーク** を守る
- 参加者は **管理会社** 役、大学側が **顧客・ユーザ・経営層・外部通報者** 役として電話で連絡
- 警察・IPA・個情委など **外部機関への通報は本大会では扱わない**
- 連絡手段の中心は **電話**。電話応対の良し悪しが採点に直結する

## 1件のインシデントで使う順番

```
電話が鳴る
  ↓
[01] 受電ヒアリングシート      ← 電話応対係が聞き取り、1件1シート
  ↓
[02] 技術担当引き継ぎテンプレート ← 記録係が整理、指揮係が技術依頼を明記
  ↓
[02.1] 先にやること            ← 再現→環境→構成→ログで切り分け（DDoS 判定前の必須ステップ）
  ↓
[03] シナリオ別対応プレイブック  ← 技術調査係が該当シナリオで対応
  ↓    （判定 → 封じ込め → 調査 → 復旧）
[06] DDoS 対応詳細（該当時のみ）  ← シナリオ4の深掘り
  ↓
[04] 完了報告テンプレート       ← 電話応対係が相手別に連絡
  ↓
[05] 全体チェックシート         ← 指揮係が Phase 0〜6 を通して管理
```

## チーム内の役割分担（推奨）

| 役割 | 担当 | 主な道具 |
|---|---|---|
| 電話応対係 | 1名 | 01, 04 |
| 記録係 | 1名 | 02（時系列をホワイトボードに） |
| 技術調査係 | 1名 | 03, NW機器／サーバ実機 |
| 指揮・意思決定係 | 1名 | 05, 全ファイルを俯瞰 |

> 同時に複数インシデントが来たら、**記録係が2件目の応対を応援**、指揮係は全体の優先度を管理。

## 運用のコツ

1. **時刻を全てに付ける**（mm/dd HH:MM）。 曖昧語（さっき・いま・後で）は禁止。
2. **事実と申告を分ける**。「ユーザ申告」と「観測事実」を混ぜない。
3. **封じ込めを復旧より優先**。経路を塞がずに戻すと再発する。
4. **断定を急がない**。「ランサムです」ではなく「ランサム疑いの症状」。
5. **連絡窓口は一本化**。技術担当から顧客に直接話させない。
6. **陽動に注意**。1件に集中している間に2件目が来る前提で警戒。
7. **記録は共有の場所に**。個人メモではなくチームのボード／共有ドキュメントへ。
