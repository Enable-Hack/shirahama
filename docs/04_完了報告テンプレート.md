# 04. 完了報告テンプレート

> 技術対応が完了し、顧客・ユーザに **完了の連絡** を入れるときのテンプレート集。
> 電話口で読み上げる、メールで送る、ホワイトボード掲示するなど、用途別に用意。

---

## 使い方

1. 技術担当から「対応完了」の報告を受ける
2. 完了条件が満たされていることを **チェックリスト** で確認
3. 相手（顧客経営層／顧客担当者／ユーザ）に応じて文面を選択
4. 電話 or メールで連絡、送信時刻を記録
5. 最終記録を [05_全体チェックシート] に残す

---

## 完了判定チェックリスト（これが揃うまで「完了」と言わない）

```
[ ] インシデントの **直接原因** が特定できた
[ ] **拡大・再発しない状態** になっている（封じ込め完了）
[ ] 影響範囲（利用者・データ・システム）が確定した
[ ] 影響を受けたサービスが **正常稼働** に戻った
[ ] 残タスク（監視強化・再発防止策）が整理されている
[ ] 顧客・ユーザの **追加質問に答える準備** ができている
[ ] 本対応の **記録（時系列・判断・根拠）** が残っている
```

> ひとつでも欠けている場合は「完了」と言わず「**一次対応完了**」で報告する。

---

## テンプレ 1：顧客担当者 向け（電話・短文）

> 顧客の情シス担当・窓口担当など、技術的にある程度わかる相手。

```
お世話になっております、管理会社の〇〇です。
先ほど（mm/dd HH:MM）ご連絡いただいた件について、対応が完了しましたのでご報告します。

【発生事象】
　〇〇（例：社内ポータル web01 への不正アクセスによる改ざん）

【原因】
　〇〇（例：WordPress プラグイン〇〇の脆弱性を突かれたものと推定）

【実施した対応】
　① 10:30 公開サイトをメンテナンスモードに切替
　② 10:45 不正追加された管理者アカウントを削除、管理者パスワードを一斉リセット
　③ 11:10 バックアップからWebコンテンツを復元
　④ 11:20 プラグインを最新版に更新し、公開再開

【影響範囲】
　・閲覧停止：10:30〜11:20（約50分）
　・情報漏えい：現時点で確認されず（認証ログ・アクセスログを確認済み）

【今後のお願い】
　・管理画面へのアクセス元IP制限を追加導入したく、〇日までにご検討ください
　・類似システムについても同様の脆弱性確認を予定しています

ご不明な点がありましたら、この番号にご連絡ください。
詳細の報告書は〇〇までにメールで改めてお送りします。
```

---

## テンプレ 2：顧客経営層 向け（電話・要点のみ）

> 技術詳細ではなく **判断に必要な事実** を短く。長く話さない。

```
〇〇様、先ほどの件、対応が完了しましたのでご報告します。

【事実】
　〇〇（例：サーバへの不正アクセスと公開ページ改ざん）が発生し、11:20 に復旧しました。

【影響】
　公開サイトの閲覧が約50分停止しました。
　お客様情報の外部流出は、現時点のログ確認の範囲では認められていません。

【対応】
　侵入経路を特定のうえ塞ぎ、バックアップから復元しました。

【今後】
　本日中に詳細レポートをお届けします。
　再発防止のため、システム構成の見直しを〇日までにご提案します。

ご判断が必要な事項は現時点ではございません。
何かお気づきの点があれば、ご連絡ください。
```

### 経営層対応のコツ

- **「結論→事実→対応→今後」の順**（技術的説明から入らない）
- **数字で伝える**（停止時間、影響者数、完了時刻）
- **「現時点で確認できる範囲で」** を添える（後日覆る可能性への布石）
- 技術的な詳細質問が来たら **「担当から改めてご説明します」** で折り返す

---

## テンプレ 3：一般ユーザ 向け（電話・わかりやすく）

> 技術用語を避け、**何が元に戻ったか** を中心に。

```
〇〇様、先ほどの件でお電話しました、管理会社の〇〇です。
先ほどご連絡いただいた〇〇の件、無事に解消しましたのでお知らせします。

【ご対応のお願い】
　・一度、該当の端末を再起動してください
　・再起動後、パスワードの再入力を求められたら、ご自身で新しいパスワードを設定してください
　・もし再起動後も同じ症状が出たら、すぐにこの番号にお電話ください

【お手数をおかけしたこと】
　お預かりしていたファイル／作業は、〇〇の状態に戻っています。
　ご入力いただいた情報の外部流出は、現時点では確認されていません。

何かおかしいと感じたら、ご自身で判断せずまずお電話ください。
```

### ユーザ対応のコツ

- **技術用語を使わない**（「暗号化」→「開けない状態」、「セッション切断」→「ログインし直し」）
- **次にやってもらう作業は1〜2個まで**
- **「大丈夫です」の代わりに「現時点で◯◯は確認されていません」**
- 不安を煽らず、かつ 油断させない

---

## テンプレ 4：外部第三者通報者 向け（電話・簡潔）

> 「貴社から不審メールが…」など通報してくれた相手。詳細は最小限に。

```
〇〇様、先日ご連絡いただいた件でお電話しました、〇〇です。
ご通報いただき、ありがとうございました。

弊社内で確認したところ、〇〇（例：社内アカウントの一部が悪用され、ご迷惑をおかけしたこと）を確認しました。
すでに該当アカウントを停止し、送信経路を遮断しています。

現時点で追加の被害拡大は確認されておりません。
もし同種のメールが再度届くようでしたら、お手数ですが弊社までご連絡ください。

改めて、ご通報いただきありがとうございました。
```

### 外部対応のコツ

- **社内事情の詳細は開示しない**（通報者が利害関係者とは限らない）
- **感謝を必ず伝える**
- **追加情報の提供ルート** を残す

## 「一次対応完了」報告テンプレ（まだ完全復旧ではない場合）

> 封じ込めはできたが、原因特定や完全復旧は継続中のとき。

```
〇〇の件、**一次対応** が完了しましたのでご報告します。

【現在の状態】
　・拡大は停止（封じ込め完了）
　・サービスは〇〇の状態で稼働（例：一部機能制限、読み取り専用）

【継続中の作業】
　・〇〇（例：侵入経路の詳細特定）
　・〇〇（例：バックアップからの完全復元）

【想定完了時刻】
　〇〇（根拠：〇〇）

【次回ご報告時刻】
　mm/dd HH:MM

完全復旧まで監視を継続します。
```

---

## 報告時の時系列記載の注意

> 完了報告では **時系列を正確に** 書く。記憶ではなく、チームの記録から転記する。

- 相手の認識時刻 vs こちらの観測時刻を分ける
- 重要な判断・封じ込め・復旧時刻は **必ず記載**
- 未解決・調査中の項目は **「調査継続中」** と明記、隠さない
- 根拠データ（ログ、画面キャプチャ等）の所在を記しておく（後日の問い合わせ対応のため）

---

## 完了報告で言ってはいけないこと

- 「**もう絶対に大丈夫です**」（未来保証はできない）
- 「**原因は完全に分かりました**」（完全に分かるケースは稀。根拠を添える）
- 「**ご迷惑をおかけしました、担当者を処分します**」（空約束・責任転嫁）
- 「**社内でもこんなことは初めてで…**」（組織の落ち度を強調する言い方）
- 未確認のまま「**漏えいはありません**」（必ず「現時点で確認できる範囲では」を付ける）

---

## 完了報告後にやること（チーム内）

```
[ ] 対応記録を事案ID単位で1箇所にまとめる
[ ] 時系列・判断・根拠をレポート形式で整理
[ ] 技術担当と共有：再発防止の技術タスクを洗い出し
[ ] チーム振り返り（KPT／よかった点・悪かった点・次回へ）
[ ] 顧客向けの詳細レポート（必要な場合）の作成予定を立てる
```
